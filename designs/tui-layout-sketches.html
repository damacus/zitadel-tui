Zitadel TUI migration sketches

Three non-tabbed directions.

Each concept keeps the same Ratatui-friendly shell discipline from tui-expert, but avoids the GitLab-style tab dashboard. These are operator-first layouts: stable regions, strong focus, and room for secrets, warnings, and deliberate admin actions.

01 Control Matrix 02 Atelier Workspace 03 Ledger Explorer 04 Hybrid Direction

How to evaluate

  • Navigation weight How much screen real estate is dedicated to moving between tasks and resources.
  • Mutation safety Whether destructive or credential-heavy actions feel deliberate instead of twitchy.
  • Terminal adaptability How cleanly the concept can collapse from a wide desktop terminal to a narrower shell.
  • Headless mental model Whether the interactive layout aligns cleanly with the CLI command groups and JSON actions.
Concept 01

Control Matrix

A command browser with a sharp, technical feel. This is the most direct replacement for prompt-by-prompt Ruby, but rendered as a persistent workstation: command rail, results grid, and an inspector for the selected object.

Concept 02

Atelier Workspace

Concept 03

Ledger Explorer

A sober, inventory-first layout. The left side groups resources like sections in a ledger, while the right side turns into either a list/detail explorer or a review board. This feels deliberate, durable, and a little less flashy than the other two.

Concept 04

Command Atelier

This is the blend: Control Matrix on the edges, Atelier in the middle. The shell keeps a disciplined command rail, but the working area becomes a deliberate canvas for forms, reviews, and setup flows. The inspector moves out of the permanent layout and opens as a focused popup.
